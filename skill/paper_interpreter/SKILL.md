# 论文解读技能

## 核心目标

**为背景知识薄弱的读者解读论文难懂之处。**

解读报告不是翻译或摘要，而是：
- 🎯 解释论文中的"黑话"和术语
- 📚 补充必要的背景知识
- 🔍 拆解复杂公式的含义
- 💡 用通俗语言说明核心思想
- 🖼️ 保留原论文的所有关键图片

## 解读六则

### 1. 术语白话化

每个专业术语要转化为通俗解释：

| 论文术语 | 解读方式 |
|---------|---------| 
| "自回归解码" | "一个字一个字地生成，每次生成都依赖前面的所有字" |
| "残差分布" | "目标模型想要但草稿模型没猜到的那些词的概率分布" |
| "KV缓存" | "把之前计算过的结果存起来，避免重复计算" |

### 2. 公式拆解

每个重要公式都要说明：
- **这个公式在算什么？**
- **每个符号代表什么？**
- **直觉上怎么理解？**

使用 `equationbox` 环境包裹重要公式。

### 3. 背景知识补充

遇到专业概念时，用 `conceptbox` 环境补充背景：
```latex
\begin{conceptbox}[背景知识：推测解码]
  想象你在写作文：
  \begin{itemize}
    \item \textbf{普通方式}：每写一个字都要仔细思考，很慢
    \item \textbf{推测解码}：先快速写个草稿，然后一次性检查修改
  \end{itemize}
\end{conceptbox}
```

### 4. 类比解释

用生活中的类比说明复杂概念。

### 5. 图文结合

**必须保留原论文的所有关键图片！**

```latex
\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/fig1.pdf}
  \caption{【原论文图1】简要描述...}
\end{figure}
```

每张图片后添加解读说明（使用 `infobox`）。

### 6. 公式与定理保真

- 保留关键公式原文（目标函数、约束、损失函数等）
- 保留核心定理/命题/引理的原始表述
- 对每个定理说明：(1) 前提条件 (2) 结论 (3) 作用
- 证明可不展开，但要说明思路主线
- **禁止**把方法部分写成纯文字摘要而忽略公式

## 解读清单

### ✅ 背景部分
- 这个研究领域在解决什么问题？
- 为什么这个问题重要？
- 现有方法有什么局限？
- 用一个类比说明问题背景

### ✅ 方法部分
- 核心思想一句话概括
- 关键术语白话解释
- 算法流程步骤列表
- 公式拆解
- 定理/命题逐条解释
- 保留原始图片并加解读

### ✅ 实验部分
- 实验验证什么假设？
- 对比方法的公平性
- 结果表格的核心结论
- 保留实验图表

## 质量自查

写完后检查：
1. ☐ 背景薄弱的同学能看懂吗？
2. ☐ 每个术语都有解释吗？
3. ☐ 每个公式都拆解了吗？
4. ☐ 所有重要图片都保留了吗？
5. ☐ 关键公式是否完整保留？
6. ☐ 定理/命题是否写明前提、结论、作用？
